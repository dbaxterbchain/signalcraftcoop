Folder guide
- docs/architecture: system design, data model, infrastructure, security
- docs/product: requirements, user flows, order lifecycle
- docs/ux: brand system, UI patterns, component guidelines
- docs/api: API contracts and OpenAPI spec
- docs/ops: CI/CD, runbooks, monitoring
- docs/decisions: architecture decision records (ADRs)

Conventions
- Update docs with every major feature or workflow change.
- Use ADRs for decisions that are hard to reverse.
- Keep docs short and link to deeper pages as needed.
